# Portfolio Insight & Archiver — 要件定義書

## Intent Analysis Summary

| 項目 | 内容 |
|---|---|
| **リクエストタイプ** | New Project（グリーンフィールド） |
| **スコープ** | System-wide（フルスタック新規開発） |
| **複雑度** | Complex（外部API連携・OAuth・Google Docs連携・Docker） |
| **ユーザー** | 個人投資家（シングルユーザー） |

---

## 1. プロジェクト概要

SBI証券からダウンロードしたポートフォリオCSVを取り込み、投資分析・アドバイス生成・Google Docsへのアーカイブを一元管理する個人向けWebアプリケーション。

バリュー株・高配当株・国策テーマ銘柄（造船・銀行・保険等）を中心とした日本株ポートフォリオの構成分析と、買い増し・整理のアドバイス生成を目的とする。

## 2. 機能要件

### 2.1 CSVインポート

| ID | 要件 |
|---|---|
| FR-CSV-01 | SBI証券からエクスポートされたCSVファイル（`New_file.csv`）を、設定されたローカルパスからサーバーサイドで直接読み込む |
| FR-CSV-02 | CSVはDockerコンテナにマウントされたディレクトリから読み込む（CLIライクな操作フロー） |
| FR-CSV-03 | CSVカラム構造: `銘柄（コード）`, `買付日`, `数量`, `取得単価`, `現在値`, `前日比`, `前日比（％）`, `損益`, `損益（％）`, `評価額` |
| FR-CSV-04 | 取り込み成功後、CSVデータをSQLiteに保存しスナップショットとして管理する |

### 2.2 外部データ取得（J-Quants API）

| ID | 要件 |
|---|---|
| FR-JQUANTS-01 | CSVに含まれる各銘柄コードを元に、J-Quants APIからメタデータを取得する |
| FR-JQUANTS-02 | 取得対象: セクター（東証33業種）、配当利回り、決算発表日、時価総額 |
| FR-JQUANTS-03 | PBR・PERは取得可能であれば取得し、オンスクリーン分析に利用する |
| FR-JQUANTS-04 | 対象市場: 日本株のみ（東証プライム・スタンダード・グロース） |

### 2.3 ポートフォリオ分析

| ID | 要件 |
|---|---|
| FR-ANALYSIS-01 | セクター別構成比を計算・表示する（東証33業種分類を使用） |
| FR-ANALYSIS-02 | セクターの偏りを可視化し、過集中しているセクターを識別する |
| FR-ANALYSIS-03 | 各銘柄のPBR・PER・配当利回りを一覧表示する（J-Quantsデータ） |
| FR-ANALYSIS-04 | 前回スナップショットとの差分を計算する（増減銘柄・評価額の変化） |
| FR-ANALYSIS-05 | 損益・損益率・評価額を集計・表示する |

### 2.4 AI分析プロンプト生成

| ID | 要件 |
|---|---|
| FR-AI-01 | ポートフォリオデータ・分析結果・投資方針（バリュー株・高配当・国策テーマ重視）を組み込んだAI分析プロンプトを自動生成する |
| FR-AI-02 | 生成されたプロンプトをユーザーがコピーし、ChatGPT・Claude等のChat UIに手動でペーストして利用する |
| FR-AI-03 | プロンプトには買い増し候補・整理候補の判断基準を含める |
| FR-AI-04 | AI APIの直接呼び出しは行わない（コスト削減のため） |

### 2.5 Google Docsアーカイブ

| ID | 要件 |
|---|---|
| FR-GDOCS-01 | CSVインポートのたびに自動でスナップショットをGoogle Docsに保存する |
| FR-GDOCS-02 | Google Docのファイル名は `YYYY-MM-DD` 形式（日付ユニーク） |
| FR-GDOCS-03 | 同日に複数回インポートされた場合、既存のGoogle Docを上書きする |
| FR-GDOCS-04 | Google Driveの特定フォルダ内に新規Docを作成する（スナップショットごとに1Docを作成） |
| FR-GDOCS-05 | アーカイブに含める内容: ポートフォリオ全銘柄リスト（A）、セクター別構成比（B）、AI分析プロンプトテキスト（D）、前回スナップショットとの差分（E） |
| FR-GDOCS-06 | PBR/PER/配当利回り一覧表はGoogle Docsアーカイブに含めない（オンスクリーン分析のみに利用） |

### 2.6 認証・セキュリティ

| ID | 要件 |
|---|---|
| FR-AUTH-01 | Googleサービスアカウント（JSONキーファイル）を使用してGoogle Docs / Drive APIに接続する |
| FR-AUTH-02 | シングルユーザー前提（マルチユーザー対応不要） |
| FR-AUTH-03 | JSONキーファイルパス（`GOOGLE_SA_KEY_PATH`）とJ-Quantsトークン（`JQUANTS_REFRESH_TOKEN`）は環境変数で管理し、SQLiteには保存しない |
| FR-AUTH-04 | ユーザーはサービスアカウントのメールアドレスにGoogle Driveフォルダの編集権限を付与する（初回セットアップのみ） |

## 3. 非機能要件

### 3.1 パフォーマンス

| ID | 要件 |
|---|---|
| NFR-PERF-01 | CSVインポート〜分析完了のレスポンスタイムは10秒以内（外部API呼び出し込み） |
| NFR-PERF-02 | J-Quants APIのレスポンスが遅い場合はタイムアウト処理を実装し、部分的なデータでも表示する |
| NFR-PERF-03 | J-Quantsデータは一定期間（例: 24時間）キャッシュし、不要な外部API呼び出しを削減する |

### 3.2 セキュリティ（Security Baseline 適用）

| ID | 要件 |
|---|---|
| NFR-SEC-01 | SECURITY-01: SQLite（ローカル）はファイルシステムレベルでの暗号化対応、通信はTLS/HTTPS必須 |
| NFR-SEC-03 | SECURITY-03: 構造化ログ（timestamp・ログレベル・メッセージ）を実装。機密情報（トークン・API Key）はログ出力しない |
| NFR-SEC-04 | SECURITY-04: 全HTML提供エンドポイントにHTTPセキュリティヘッダーを設定（CSP・HSTS・X-Content-Type-Options等） |
| NFR-SEC-05 | SECURITY-05: 全APIエンドポイントで入力バリデーション実施（型・長さ・フォーマット・インジェクション防止） |
| NFR-SEC-08 | SECURITY-08: 全エンドポイントに認証ミドルウェアを適用。CORS設定は特定オリジンのみ許可 |
| NFR-SEC-09 | SECURITY-09: デフォルト認証情報の禁止、本番エラーレスポンスはスタックトレース非公開 |
| NFR-SEC-10 | SECURITY-10: 依存関係はロックファイルで固定、脆弱性スキャンをビルドフローに組み込む |
| NFR-SEC-11 | SECURITY-11: セキュリティクリティカルなロジック（OAuth・認証）は専用モジュールに分離 |
| NFR-SEC-12 | SECURITY-12: J-Quantsトークン・サービスアカウントJSONキーパスは環境変数のみで管理。SQLite保存禁止・ソースコードへのハードコード禁止 |
| NFR-SEC-15 | SECURITY-15: 全外部呼び出し（J-Quants API・Google API）に明示的エラーハンドリング。グローバルエラーハンドラーを実装 |

**注**: SECURITY-02（ネットワーク中間者アクセスログ）・SECURITY-06（IAMポリシー）・SECURITY-07（ネットワーク設定）・SECURITY-13（SRI）・SECURITY-14（アラート監視）はローカルDocker環境のため N/A。

| フィールド | 用途 |

## 7. 主要ユーザーシナリオ

### シナリオ 1: 定期ポートフォリオ更新

1. ユーザーがSBI証券からCSV（`New_file.csv`）をダウンロードし、マウントされたディレクトリに配置する
2. WebUIからCSVインポートを実行する
3. バックエンドが指定パスからCSVを読み込み、J-Quants APIでデータを補完する
4. 分析結果（セクター分布・評価損益・PBR/PER等）をWebUI上に表示する
5. AI分析プロンプトを生成し、コピーボタンで取得する
6. 自動でGoogle Docsにスナップショットを保存する（既存の同日Docは上書き）

### シナリオ 2: 過去スナップショットとの比較

1. WebUI上で過去のスナップショット一覧を参照する
2. 特定日のスナップショットを選択する
3. 現在または別の日付との差分（増減銘柄・評価額変化）を確認する

### シナリオ 3: 初回セットアップ（Google OAuth）

1. アプリ初回起動時にGoogle OAuthサインインを行う
2. Google Drive / Google Docs アクセス権限を承認する
3. アーカイブ先のGoogle Driveフォルダを設定する

## 8. 制約・前提条件

| 項目 | 内容 |
|---|---|
| 対象市場 | 日本株のみ（東証プライム・スタンダード・グロース） |
| CSV形式 | SBI証券のポートフォリオCSVフォーマット固定 |
| セクター分類 | 東証33業種分類を使用 |
| AI分析方式 | プロンプト生成のみ。AI APIへの直接呼び出しなし |
| 実行環境 | ローカルPC上のDocker Compose（ローカル専用） |
| ユーザー数 | シングルユーザー（個人利用） |
| J-Quants APIキー | ユーザーが個別に取得・設定 |
| Google OAuth資格情報 | ユーザーが個別にGoogle Cloud Consoleで取得・設定 |

---

## 9. 除外スコープ

- AI API（OpenAI・Anthropic・Gemini）への直接呼び出し
- 米国株・外国株の対応
- マルチユーザー対応
- クラウドデプロイ（ローカルDocker専用）
- PBR/PER/配当利回り一覧のGoogle Docsアーカイブへの保存（オンスクリーン分析のみ）
- 自動スケジューリング（手動インポートのみ）
